# Pratibimb Documentation

Pratibimb is a Chromium browser extension for Windows that syncs the browser new tab background with the current Windows desktop wallpaper and dynamically themes the UI.

## Overview

Pratibimb provides a custom new tab experience that:

- Uses Chromium native messaging to fetch the current Windows wallpaper.
- Displays a polished new tab page with a live clock, greeting, weather, and search bar.
- Applies a dynamic color palette based on the wallpaper image for a cohesive look.
- Caches the wallpaper and palette in `localStorage` for instant reloads.

## Components

### Browser extension (`extension/`)

- `manifest.json`
  - Chrome Manifest V3 configuration.
  - Overrides the browser new tab page with `newtab.html`.
  - Requests `nativeMessaging`, `storage`, and `unlimitedStorage` permissions.
  - Declares host permissions for weather, reverse geocoding, and Google search suggestions.

- `newtab.html`
  - Defines the HTML structure for the custom new tab page.
  - Loads fonts and inline CSS for styling.
  - Includes scripts:
    - `fastload.js`
    - `ui.js`
    - `newtab.js`

- `fastload.js`
  - Loads cached wallpaper and CSS palette from `localStorage` instantly.
  - Ensures the page looks populated while the native host is still responding.

- `ui.js`
  - Connects to the native host using `chrome.runtime.connectNative('com.suntzv.pratibimb')`.
  - Receives wallpaper image chunks from the host and reconstructs them.
  - Converts them into a `data:image/jpeg;base64` URL.
  - Computes a dynamic accent palette from a small version of the wallpaper.
  - Stores `instantWallpaper` and `dynamicPalette` for future fast loads.

- `newtab.js`
  - Renders the live clock, formatted date, and greeting.
  - Retrieves weather data from Open-Meteo and reverse geocodes location via Nominatim.
  - Caches weather display for 30 minutes.
  - Implements Google search suggestions with debounced input.
  - Provides keyboard navigation and focus behavior for search.

### Native host (`host/`)

- `main.cpp`
  - Native Windows host application for native messaging.
  - Reads the active wallpaper file from:
    - `%APPDATA%\Microsoft\Windows\Themes\TranscodedWallpaper`
  - Uses `stb_image.h` and `stb_image_write.h` to decode and re-encode the wallpaper.
  - Encodes the final wallpaper image to JPEG then Base64.
  - Sends the encoded image in JSON chunk messages over stdin/stdout.
  - Handles failure by returning an empty result payload.

- `com.suntzv.pratibimb.json`
  - Native messaging manifest declaring the host path and allowed extension origin.
  - Should be generated by the install script to preserve correct executable path.

## Installation

### Build the host application

Compile `host/main.cpp` into `host/pratibimb_host.exe`.

The extension installation expects this executable to exist at:

- `host/pratibimb_host.exe`

### Install native host and extension support

Run the installer executable directly:

- `install\pratibimb_installer.exe`

If `install\pratibimb_installer.exe` is not present, compile `install\installer.cpp` locally or ship a prebuilt `install\pratibimb_installer.exe` so end users do not need a compiler.

The repository ignores compiled binaries and extension packaging files, so you may keep `install\pratibimb_installer.exe`, `extension.crx`, and `extension.pem` locally for packaging without checking them into Git.

`install\installer.exe` provides a clean GUI that can:

- Install the native messaging backend.
- Uninstall the native messaging backend.
- Register the native host for supported Chromium browsers.
- Optionally install the extension into selected browsers by path. This can be either a local unpacked extension folder or a `.crx` package, so users do not need to enable developer mode.

Supported browsers include:

- Google Chrome
- Brave
- Microsoft Edge
- Chromium
- Opera
- Vivaldi

### Uninstall

Use the `Uninstall` option inside `install\pratibimb_installer.exe` to remove native host registration and extension entries from selected browsers.

## Runtime flow

1. The browser opens the custom `newtab.html` page.
2. `fastload.js` immediately applies cached wallpaper and palette if available.
3. `ui.js` connects to the native host `com.suntzv.pratibimb`.
4. The host reads the Windows wallpaper file and sends Base64 chunks.
5. `ui.js` reconstructs the image, stores it, and updates the page background.
6. `newtab.js` updates clock, weather, greeting, and search suggestion UI.

## Notes

- The project is designed for Chromium-based browsers on Windows.
- Native messaging requires registry registration for the browser.
- The wallpaper source is the Windows theme wallpaper file.
- `com.suntzv.pratibimb.json` is generated dynamically by the installer.

> The native messaging manifest is bound to the extension origin. If the extension ID differs in a particular browser installation, update `allowed_origins` accordingly.
## Troubleshooting

- If the wallpaper doesn't load, verify the native host executable exists and the manifest path is correct.
- Confirm the registry key is present under the browser you are using, for example:
  - `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.suntzv.pratibimb`
  - `HKCU\Software\BraveSoftware\Brave-Browser\NativeMessagingHosts\com.suntzv.pratibimb`
  - `HKCU\Software\Microsoft\Edge\NativeMessagingHosts\com.suntzv.pratibimb`
  - `HKCU\Software\Chromium\NativeMessagingHosts\com.suntzv.pratibimb`
  - `HKCU\Software\Opera Software\Opera Stable\NativeMessagingHosts\com.suntzv.pratibimb`
  - `HKCU\Software\Vivaldi\Vivaldi\NativeMessagingHosts\com.suntzv.pratibimb`
- Make sure the extension ID in the native manifest matches the actual installed extension.

## Development

- Modify `newtab.html` for layout updates.
- Update `newtab.js` for search, weather, or clock logic.
- Adjust `ui.js` for theme extraction and native host data processing.
- Change `host/main.cpp` to support different wallpaper sources or encoding behavior.
