# Pratibimb Documentation

***

Pratibimb is a Chromium browser extension for Windows that syncs the browser new tab background with the current Windows desktop wallpaper and dynamically themes the UI.

## Overview

Pratibimb provides a custom new tab experience that:
- Uses Chromium native messaging to fetch the current Windows wallpaper.
- Displays a polished new tab page with a live clock, greeting, weather, and search bar.
- Applies a dynamic color palette based on the wallpaper image for a cohesive look.
- Caches the wallpaper and palette in `localStorage` for instant reloads.

## Components

### Browser extension (`extension/` or extracted from `extension.zip`)
- `manifest.json`
  - Chrome Manifest V3 configuration.
  - Overrides the browser new tab page with `newtab.html`.
  - Requests `nativeMessaging`, `storage`, and `unlimitedStorage` permissions.
  - Declares host permissions for weather, reverse geocoding, and Google search suggestions.

- `newtab.html` & Scripts (`fastload.js`, `ui.js`, `newtab.js`)
  - Loads cached wallpaper and CSS palette from `localStorage` instantly.
  - Connects to the native host using `chrome.runtime.connectNative('com.suntzv.pratibimb')`.
  - Reconstructs Base64 image chunks into a dynamic accent palette and background.
  - Handles Open-Meteo caching, Nominatim reverse geocoding, and debounced Google search.

### Native host (`host/`)
- `main.cpp`
  - Native Windows host application for native messaging.
  - Reads the active wallpaper file from `%APPDATA%\Microsoft\Windows\Themes\TranscodedWallpaper`.
  - Uses `stb_image.h` and `stb_image_write.h` to decode and encode to JPEG/Base64.
  - Sends the encoded image in JSON chunk messages over stdin/stdout.

- `com.suntzv.pratibimb.json`
  - Native messaging manifest. **This is dynamically generated by the installer** to ensure it contains the exact executable path and the specific Extension ID generated by the user's browser.

### Monolithic Installer (`install/`)
- `installer.cpp` & `resources.rc`
  - A standalone, zero-dependency C++ GUI using the Windows Desktop Window Manager (DWM) for immersive dark mode.
  - Packs the compiled `pratibimb_host.exe` and `extension.zip` directly into the `.exe` binary via `resources.res`.
  - Extracts payloads to `%LOCALAPPDATA%\Pratibimb` and silently executes PowerShell to unpack the extension directory.
  - Takes a user-provided Extension ID, generates the Native Manifest, and writes the required registry keys across all supported Chromium browsers.

## Build Pipeline (For Developers)

To compile the project from source, you need the **MinGW-w64 (GCC)** toolchain.

1. **Build the Backend:** Compile `host/main.cpp` into `host/pratibimb_host.exe`.
2. **Zip the Frontend:** Select all files *inside* your `extension` folder and compress them into `extension.zip` in the project root.
3. **Compile Resources:** Open a terminal in the `install/` directory and pack the payloads into a binary resource file:
   ```powershell
   windres resources.rc -O coff -o resources.res
   ```
4. **Compile the Installer:** Compile the GUI installer, linking all necessary libraries statically:
   ```powershell
   g++ -std=c++17 -O2 installer.cpp resources.res -o Pratibimb_Setup.exe -mwindows -municode -lcomctl32 -lole32 -ladvapi32 -lshell32 -ldwmapi -luuid -static -static-libgcc -static-libstdc++
   ```

## Installation Workflow (End-User)

Because modern Chromium security strictly blocks automated registry installations that override the "New Tab" page, Pratibimb uses a secure hybrid installation.

1. **Extract:** Run `Pratibimb_Setup.exe` and click **1. Extract & Open Folder**. The installer unpacks the extension into `%LOCALAPPDATA%\Pratibimb\extension` and opens the folder.
2. **Load Unpacked:** The user opens their browser's extension page (e.g., `edge://extensions`), enables **Developer Mode**, and drags the extracted folder into the browser.
3. **Link:** The browser generates a unique Extension ID based on the folder path. The user copies this ID, pastes it into the installer, and clicks **3. Link Native Host**. The installer binds the backend to that specific ID via the registry.

## Uninstall

Use the **Uninstall** radio button inside `Pratibimb_Setup.exe`. This performs a clean wipe:
- Recursively deletes `%LOCALAPPDATA%\Pratibimb`.
- Removes `com.suntzv.pratibimb` registry entries from all supported browsers.
- Leaves zero trace on the filesystem.

## Notes & Chromium Security Limits

- **The New Tab Hard-Block:** Local `.crx` files sideloaded via the registry are permanently disabled by Chromium if they attempt to override the New Tab page. The "Load Unpacked" folder method is the only way to bypass this locally without publishing to a Web Store.
- **Dynamic IDs:** When an extension is loaded as a raw folder, the browser calculates its ID based on its absolute file path. If the `%LOCALAPPDATA%\Pratibimb\extension` folder is ever moved or renamed, the browser will generate a *new* ID, which will instantly break the Native Host connection until re-linked via the installer.

## Troubleshooting

- **"The Native Host disconnected or isn't responding."**
  - Verify the Extension ID in the browser exactly matches the `allowed_origins` string in `%LOCALAPPDATA%\Pratibimb\host\com.suntzv.pratibimb.json`.
- **"Installer fails to unpack."**
  - Ensure `resources.res` was successfully compiled and linked during the build process. If `extension.zip` or `pratibimb_host.exe` are missing from the build directory when `windres` runs, the installer will compile but be empty.
- **Registry Keys missing:**
  - Verify the keys are being written to `HKCU\Software\[Browser]\NativeMessagingHosts\com.suntzv.pratibimb`.
